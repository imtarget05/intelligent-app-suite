# Contributing to Awesome LLM Apps

Thank you for your interest in contributing.
This repository is a collection of practical AI Agent and RAG templates, so changes should stay runnable and easy to understand.

## Ways to Contribute

- Report bugs
- Suggest new templates or improvements
- Improve documentation
- Submit bug fixes or new features

## Before You Start

1. Search existing issues and pull requests to avoid duplicates.
2. Open an issue first for large changes, architecture updates, or new app proposals.
3. Keep pull requests focused and scoped to one topic.

## Development Guidelines

1. Fork the repository and create a feature branch.
2. Make changes in the relevant project folder.
3. Keep setup instructions accurate (`requirements.txt`, `.env.example`, run commands).
4. Do not commit secrets (API keys, tokens, credentials).
5. Update docs when behavior or setup changes.

## Pull Request Checklist

Before submitting, confirm:

- [ ] Code runs locally
- [ ] README or docs are updated (if needed)
- [ ] No secrets or sensitive data are included
- [ ] Changes are limited to the intended scope
- [ ] PR description explains what changed and why

## Commit Messages

Use clear, descriptive commit messages.
Examples:

- `fix: resolve streamlit startup error in local_rag_agent`
- `docs: add setup steps for gemini_agentic_rag`
- `feat: add citation support to rag_chain example`

## Review Process

- Maintainers review pull requests for correctness, clarity, and repo consistency.
- Feedback may request revisions before merge.
- Keep discussion in the PR thread for traceability.

## Need Help?

If you are unsure where to start, open an issue with context and expected outcome.
